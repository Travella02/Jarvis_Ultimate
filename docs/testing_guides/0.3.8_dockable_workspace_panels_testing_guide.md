# Testing Guide — 0.3.8 Dockable Workspace Panels

Run from the Jarvis project root after applying the patch.

```powershell
python -m unittest discover -s tests -v
python scripts\start_jarvis_app.py
```

Manual UI checks:

1. Start Jarvis.
2. Use the `Layout: Unlocked` button to make sure layout editing is enabled.
3. Drag a panel by its heading.
4. Resize a panel from the lower-right corner.
5. Restart Jarvis and confirm the panel stays where you placed it.
6. Click `Layout: Locked` and confirm accidental dragging is prevented.
7. Try `Gaming`, `Coding`, `Music`, and `Minimal` from the layout preset dropdown.
8. Type a panel name into `Open panel...`, such as `chat`, `voice`, or `workspace`, and press Enter.
9. Click `Pop` on a panel and move the popout window to another monitor.
10. Click `Dock` to return a panel to the main layout.
11. Click `Reset Layout` if anything looks wrong.

Expected result: panels should be movable, resizable, persistent, resettable, and pop-out capable without breaking Jarvis voice/chat behavior.
