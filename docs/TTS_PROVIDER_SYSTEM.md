# TTS Provider System

Version: 0.0.7

Jarvis voice output is built as a swappable provider layer. The brain should not
know whether speech is generated by XTTS, Kokoro, ElevenLabs, Windows TTS, or a
mock provider. The TTS manager owns provider selection, fallback, output files,
and optional playback.

## Current provider strategy

- `xtts` is the preferred provider for Tanner's current personal/local Jarvis
  voice experiments because it can support voice cloning from a reference WAV.
- `kokoro` is the local fallback and the safer direction for a future SaaS-style
  product because the Kokoro model/weights are Apache-licensed.
- `mock` is always the final fallback so tests and fresh installs still work
  without heavy TTS dependencies.

## SaaS/commercial licensing note

XTTS v2's public model license is non-commercial and covers model outputs. Keep
XTTS for personal local testing only unless a separate commercial license is
secured. If Jarvis becomes SaaS/commercial, switch the default provider to
Kokoro, ElevenLabs, or another provider with clear commercial rights before
shipping.

## Configuration

The main settings live in `.env` or `config/providers.yaml`:

```env
JARVIS_TTS_ENABLED=true
JARVIS_TTS_PROVIDER=xtts
JARVIS_TTS_FALLBACK_PROVIDERS=kokoro,mock
JARVIS_TTS_OUTPUT_DIR=data/tts
JARVIS_TTS_VOICE_NAME=jarvis
JARVIS_TTS_LANGUAGE=en
JARVIS_TTS_DEVICE=cuda
JARVIS_TTS_USE_GPU=true
JARVIS_TTS_PLAYBACK=false
JARVIS_TTS_AUTO_SPEAK=false
JARVIS_TTS_XTTS_MODEL_NAME=tts_models/multilingual/multi-dataset/xtts_v2
JARVIS_TTS_XTTS_SPEAKER_WAV=assets/voices/jarvis_reference.wav
JARVIS_TTS_KOKORO_VOICE=af_heart
JARVIS_TTS_KOKORO_LANG_CODE=a
```

## CLI commands

```text
tts status
tts providers
tts test
tts say Hello Tanner. Jarvis voice output is connected.
voice on
voice off
```

`voice on` only enables automatic speech for the current CLI runtime session. It
does not change `.env`.

## Optional dependency installation

The main Jarvis test suite does not require XTTS or Kokoro. When ready to test
real local speech, install optional dependencies from:

```powershell
pip install -r requirements-tts.txt
```

XTTS can be sensitive to Python, PyTorch, and CUDA versions. If installation is
rough, keep the provider system and use `mock` or `kokoro` until the GPU TTS
environment is stable.

## Reference voice file

For XTTS voice cloning, place a clean WAV file at:

```text
assets/voices/jarvis_reference.wav
```

A short, clean, dry voice sample usually works better than a noisy or reverbed
sample. Do not use someone else's voice without permission.
