# UI and Avatar System

Jarvis Core should be able to run without the UI.

The desktop UI is a client that connects to Jarvis Core. The avatar/body reacts to events from the core.

Example events:
- brain.thinking_started
- brain.thinking_finished
- voice.speaking_started
- voice.speaking_finished
- agent.screen_agent.active
- recording.started
- recording.stopped
- error.detected

The avatar should not be the brain. It is Jarvis's visual body.
